Barometer

This fun little vulnerability uses machine learning to break into a locked smartphone using nothing but the motion sensors.

Here's how it works.
(also pls use a desktop/laptop/horizontal-screened-device)

Scroll

The attack starts with a mobile game. We'll use Whack-A-Mole!

The user has no idea this innocent-looking game is secretly recording all the data it can get.
This includes: touch data,
...air pressure data,
...and gyroscope motion sensor data.
Notice how every time the user taps, there's a spike in the pressure sensor?

The game developer hacker can take advantage of this when the user is later typing their PIN!
For example, I was easily able to train a windowed regression model on the air pressure signal.

This can then be thresholded to blindly figure out when the user taps!
This gets us 50% of the way.

Next, we can use the gyroscope data to figure out how the user taps.
I have yet to document the full process of this, but if you're curious, you can read these academic research papers:
  • (2012) Practicality of Accelerometer Side Channels on Smartphones
  • (2020) AlphaLogger: detecting motion‑based side‑channel attack using smartphone keystrokes
  • (2020) MAGLeak: A Learning-Based Side-Channel Attack for Password Recognition With Multiple Sensors in IIoT Environment
  • (2023) Machine Learning for PIN Side-Channel Attacks Based on Smartphone Motion Sensors
And if you're worried about this side-channel, you can relax. Google and Apple have been aware of this since around 2012, so this won't be exploited anytime soon on your smartphone.

But if future companies build futuristic products, but they forget to lock down access to the motion sensors, this is how they could be hacked.
